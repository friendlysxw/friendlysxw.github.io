hexo极速搭建个人博客

工具 hexo git 2018/11/24

hexo快速搭建流程

1.环境搭建及hexo框架安装

需要Node.js 环境、Git 环境以及 Hexo ,如果你尚未安装或者不了解 Hexo，请参考 hexo官方文档 进行了解以及安装。

2,修改根目录下_config.yml全局配置文件,修改其中主要配置项

#====start与=====end之间为主要配置项

#主要配置项（重要）=======其它要自己多研究了=======start

 #站点基础配置
title: TQK                      #站点标题
subtitle: 听曲客的个人博客           #站点副标题
description: 一个IT小白的成长笔记  #站点描述
keywords: 前端,后端,全栈,php       #站点关键字
author: 听曲客                    #站点作者
language: zh-CN                   #站点语言
timezone:             #网站时区。Hexo默认使用您电脑的时区。时区列表：America/New_York.. 

 #域名访问配置 
url:         #(没有自定义域名: GitHub Pages给你的域名),(有自定义域名: 域名/博客的项目名)
root:        #(没有自定义域名: 项目名称),(有自定义域名: /)

#主要配置项================================end

#文章链接地址格式 。即文章存放的目录。
permalink: :year/:month/:day/:title/
permalink_defaults:

# 目录设置
#资源文件夹，放在里面的文件会上传到github中
source_dir: source
#公共文件夹，存放生成的静态文件
public_dir: public
#标签文件夹，默认是tags。实际存放在source/tags中。
tag_dir: tags
#档案文件夹，默认是archives。
archive_dir: archives
#分类文件夹，默认是categories。实际存放在source/categories中。
category_dir: categories
#代码文件夹，默认是downloads/code
code_dir: downloads/code
#国际化文件夹，默认跟language相同
i18n_dir: :lang

#不需要渲染的文件夹或文件夹,放在[]中
skip_render: [README.md,...]

# 写作选项
# 新建博文（帖子）的默认名称
new_post_name: :title.md 
#默认布局模板是post，而不是draft和page
default_layout: post
#是否将标题转换成标题形式（首字母大写）
titlecase: false 
#在新标签页面中打开网页
external_link: true 
filename_case: 0
#是否渲染草稿
render_drafts: false
#是否启动 Asset 文件夹
post_asset_folder: false
#把链接改为与根目录的相对位址
relative_link: false

#显示未来的文章
future: true
highlight:
  enable: true        #  使用代码高亮
  line_number: true   # 显示行号
  auto_detect: true   # 自动检测语言
  tab_replace:

#主要配置项================================start
#项目部署时(hexo d时)的首页渲染配置，【可不改动】
index_generator:    
  path: ''     #项目部署时首页文件生成路径，默认为空,则在项目根目录创建index.html文件
              #比如想在主题首页之前再加个引导页,可修改此配置,指定将主题首页放在其它目录,
              #如path:'hexo_index'就会在站点根目录创建hexo_index文件夹,并在其下创建index.html
              #这样访问网站时就不会显示主题特定的首页，而显示自定义首页
              #自定义首页可在 ‘根目录/themes/主题根目录/source/’ 下自行创建index.html

  per_page: 10        #首页显示列表分页大小
  order_by: -date   #时间倒序排序
#主要配置项================================end

# 分类和标签
# 默认分类
default_category: uncategorized
#分类别名
category_map:
#标签别名
tag_map:

# 日期和时间格式
#Hexo 使用 Moment.js 来解析和显示时间。
date_format: YYYY-MM-DD
time_format: HH:mm:ss

# 分页配置
per_page: 10
pagination_dir: page

#主要配置项(重要)================================start
#选择网站用的主题名称(themes/主题文件夹名称)
theme: landscape #默认的    

#配置博客项目部署
deploy:
  type: git        #指定平台(git,coding,,,,)
  repo: https://github.com/friendlysxw/hexo_blog_tqk.git    #指定项目路径
  branch: master   #指定要部署的分支
#主要配置项================================end

3.下载主题

有两种方式获取本主题–下载 *.zip 文件和通过 git方式：

  1. 下载 主题文件解压后放在 themes 目录下，和博客中的landscape为同级目录

  2. Git方式，在 themes 目录执行：

git clone 主题git地址

4.配置主题

不同主题，配置各有不同，请根据主题发布者提供的文档说明配置相应内容

5.创建页面

hexo new md文件名称  #创建使用markdown随便编辑内容

6.编译页面

hexo g | generate    #把markdown文件编译成html文件

或

hexo generate --watch  #Hexo 能够监视文件变动并立即重新生成静态文件，在生成时会比对文件的 SHA1                          checksum，只有变动的文件才会写入。

7.本地测试

hexo s | server

8.部署，将博客推送到远程平台

hexo d | deploy        #部署后就可以通过域名愉快的访问网站啦
